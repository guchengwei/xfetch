# xfetch docs

This directory is split into:

- active docs
  - docs that describe the current operational/runtime state
- archive
  - historical plans, migration notes, and implementation records kept for reference

## Active docs

- operational-routing.md
  - the current save/publish path and routing rules for the active local workflow

## Archived docs

- archive/plans/
  - historical planning documents from the xfetch consolidation and publish-pipeline buildout

Rule of thumb:
- if a document describes how xfetch works now, keep it at the top level of docs/
- if a document mainly records a past migration, implementation milestone, or obsolete naming/state, move it under docs/archive/
